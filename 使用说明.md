# lpkx 转 lpk 工具使用说明

## 功能
将 ArcGIS Pro 的 `.lpkx` 图层包转换为 ArcMap 的 `.lpk` 图层包，尽可能保留原 lpkx 的符号化样式。

## 环境要求
- **ArcGIS Pro 的 Python 环境**：`D:\arcpy_env\python.exe`（Python 3 + arcpy）
- **ArcMap 10.8 的 Python 环境**：`C:\Python27\ArcGIS10.8\python.exe`（Python 2.7 + arcpy）
- 两个环境都需要可用，工具会自动调用

## 文件说明
| 文件 | 说明 |
|------|------|
| `lpkx转lpk工具.bat` | **主启动器**（双击运行或拖入文件） |
| `run_launcher.py` | Python 启动器（备用，编码更可靠） |
| `lpkx_to_lpk_extract.py` | Pro 端脚本（解包+样式提取） |
| `lpkx_to_lpk_build.py` | ArcMap 端脚本（重建+打包） |

## 使用方法

### 方式一：双击 bat（推荐）
1. 双击 `lpkx转lpk工具.bat`
2. 在提示后输入或拖入 `.lpkx` 文件路径，回车
3. 输入空行结束输入
4. 等待处理完成，lpk 输出到 lpkx 同目录下

### 方式二：拖入文件到 bat
1. 选中一个或多个 `.lpkx` 文件（或包含 lpkx 的文件夹）
2. 拖到 `lpkx转lpk工具.bat` 上
3. 自动批量处理

### 方式三：Python 启动器
```
D:\arcpy_env\python.exe run_launcher.py "文件路径.lpkx"
```
支持多个文件和文件夹参数。

## 输出
- `.lpk` 文件输出到 lpkx 同目录下，同名（仅扩展名不同）
- 处理报告显示成功/失败统计

## 样式保真度说明

### 完全支持的渲染器
| 渲染器类型 | 说明 | 保真度 |
|-----------|------|--------|
| SimpleRenderer（单一符号） | 颜色、轮廓、填充 | 高 |
| UniqueValueRenderer（唯一值） | 字段、值、每类颜色 | 高 |
| GraduatedColorsRenderer（分级色彩） | 字段、断点、色带 | 高 |

### 降级处理的符号
| 符号类型 | 处理方式 |
|---------|---------|
| HatchFill（网格填充） | 转为半透明实心填充（取网格线颜色） |
| PictureFill（图片填充） | 转为灰色实心填充 |
| CharacterMarker（字符标记） | 简化为圆形点符号 |

### 不支持的类型
- 栅格 Colorizer（栅格渲染器）→ 使用默认符号
- 点云/3D 符号 → 不支持
- Pro 专属符号 → 不支持

## 技术原理
1. **Pro 端**：解包 lpkx → 解析 lyrx 的 CIM JSON → 转换为 webmap JSON 格式的符号定义
2. **ArcMap 端**：加载数据 → 用 `updateLayerFromJSON` 应用符号化 → `SaveToLayerFile` → `PackageLayer` 生成 lpk

核心技术：ArcMap 的 `Layer.updateLayerFromJSON()` 方法可接受 webmap JSON 格式的图层定义（含 drawingInfo 渲染器和符号），实现精确的符号化重建。

## 注意事项
- 处理过程中会在 `%TEMP%\lpkx_to_lpk` 下创建临时文件
- 如果同目录下已存在同名 `.lpk`，会被覆盖
- ArcMap 端输出可能显示部分乱码（Python 2.7 编码限制），不影响功能
